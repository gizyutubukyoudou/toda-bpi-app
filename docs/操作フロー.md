# 火気使用届システム 操作フロー

> 最終更新: 2026-05-15

## 役職一覧

| 役職 | ロール | 説明 |
|------|--------|------|
| 協力会社（火元責任者） | contractor | 火気作業を行う会社の担当者 |
| 担当職員 | supervisor | 現場の担当職員 |
| 所長 | manager | 作業所の所長 |

---

## ワークフロー種別

本システムは **作業所単位** で2種類のワークフローを切り替えて運用できます。  
所長が「作業所設定」で選択します。

| 種別 | ステップ数 | 特徴 |
|------|-----------|------|
| **正規ワークフロー**（デフォルト） | 8ステップ | 所長事前承認 → 担当職員2回関与 |
| **簡略ワークフロー** | 5ステップ | 担当職員 or 所長どちらかで事前承認、④⑤同時操作 |

---

## 全体フロー概要

### 正規ワークフロー

```
① 申請作成・提出（協力会社）
        ↓ メール通知（担当職員・所長）
② 所長事前承認（所長）
        ↓ メール通知（協力会社）
③-1 現地確認チェック（協力会社）
        ↓ メール通知（担当職員）
③-2 担当職員直前承認（担当職員）
        ↓ メール通知（協力会社）
④-1 作業中点検サイン（協力会社）
        ↓ メール通知（担当職員）
④-2 担当職員点検サイン（担当職員）
        ↓ メール通知（協力会社）
⑤ 作業終了報告（協力会社）
        ↓ メール通知（担当職員）
⑥ 残火確認（担当職員）
        ↓ メール通知（所長）
⑦ 最終承認（所長）
        ↓ メール通知（協力会社）
✅ 承認完了
```

### 簡略ワークフロー（当現場オリジナル）

```
① 申請作成・提出（協力会社）
   ※ 提出時に現地確認チェックも同時記入
        ↓ メール通知（担当職員・所長）
① 事前承認（担当職員 または 所長）※どちらか一方で完了
        ↓ メール通知（協力会社）
④⑤ 点検・終了報告（協力会社）※作業中点検と終了報告を同一画面で同時操作
        ↓ メール通知（担当職員）
⑥ 残火確認（担当職員）
        ↓ メール通知（所長）
⑦ 最終承認（所長）
        ↓ メール通知（協力会社）
✅ 承認完了
```

> ②③（現地確認・担当職員直前承認）は提出時・①承認時に自動代替されます。

## 役職別 操作手順

---

### 協力会社（火元責任者）

#### ① 申請作成・提出

1. `https://toda-bpi-app.vercel.app` にアクセスしてログイン
2. ダッシュボード右下の **「＋」** ボタンをタップ（新規作成）
3. 以下を入力して **「下書き保存」** → **「提出」**

| 入力項目 | 内容 |
|----------|------|
| 作業所名 | 現場の作業所名（担当職員・所長と一致させること） |
| 使用日 | 火気作業を行う日付 |
| 作業時間 | 開始・終了時刻 |
| 作業場所 | 具体的な作業箇所 |
| 作業内容 | 溶接・切断・ケレン・乾燥・給熱・その他から選択＋任意テキスト |
| 火元責任者名 | 自分の氏名 |
| 作業員名 | 作業員の氏名 |
| 監視員 | 監視員の氏名・会社名 |
| 担当者 | 担当職員を複数選択（任意） |
| 火気作業の種類 | 該当する作業にチェック |
| 可燃物・作業環境・防火対策・消火設備 | 該当項目にチェック（可燃物なし選択可） |

> **前回入力の引き継ぎ**: 2回目以降の申請は前回の入力内容（チェック項目・担当者・作業場所等）が自動入力されます。変更が必要な箇所のみ修正してください。

4. 提出すると担当職員・所長にメール通知が届く

##### 簡略ワークフローの場合（追加項目）

提出フォームの下部に「現地確認チェック」が表示されます。作業直前に火元責任者が確認した上でチェックを入れて提出してください。

| 確認項目 |
|----------|
| 可燃物の除去・養生 |
| 下階の防火対策 |
| 消火設備の確認 |
| 開口部の向こう側への対策 |
| 見張り人の配置・監視手順 |
| 火気作業の表示 |

---

#### ③-1 現地確認チェック（正規ワークフローのみ・所長事前承認後）

- 所長承認のメールが届いたらアプリにログイン
- ダッシュボードの「未対応」に申請が表示される → タップ
- **「現地確認へ」** をクリック
- 以下を確認してチェックを入れ **「確認完了」**

| 確認項目 |
|----------|
| 可燃物の除去・養生 |
| 下階の防火対策 |
| 消火設備の確認 |
| 開口部の向こう側への対策 |
| 見張り人の配置・監視手順 |
| 火気作業の表示 |

---

#### ④-1 作業中点検サイン（正規ワークフローのみ・担当職員直前承認後）

- 担当職員承認のメールが届いたらアプリにログイン
- ダッシュボードの「未対応」の申請をタップ
- **「作業中点検へ」** をクリック
- 点検時刻・残火確認時刻を入力してサイン

---

#### ⑤ 作業終了報告（正規ワークフローのみ）

- 作業が終了したらアプリにログイン
- ダッシュボードの「未対応」の申請をタップ
- **「作業終了報告」** をクリック
- 作業終了時刻を入力して **「報告する」**
- 担当職員に残火確認依頼メールが届く

---

#### ④⑤ 点検・終了報告（簡略ワークフローのみ・事前承認後）

- 事前承認のメールが届いたらアプリにログイン
- ダッシュボードの「未対応」の申請をタップ
- **「点検・終了報告へ」** をクリック
- **点検時刻**と**作業終了時刻**を入力して **「⑤ 終了報告する」**
- 担当職員に残火確認依頼メールが届く

---

### 所長（manager）

#### 作業所設定（ワークフロー種別の切り替え）

1. ダッシュボード右下の **「作業所設定」** ボタンをタップ
2. ワークフローを選択
   - **正規ワークフロー**: 所長事前承認 → 現地確認 → 担当承認 → 点検終了 → 残火 → 最終
   - **簡略ワークフロー（当現場）**: 担当者or所長事前承認（現地確認同時）→ 点検終了（同時）→ 残火 → 最終
3. **「保存する」** をクリック

> この設定は今後の**新規申請**に適用されます。既存の申請中フローには影響しません。

---

#### メンバー管理（担当職員・協力会社のアカウント作成）

所長はアプリからメンバーのアカウントを作成・削除できます。

**追加**
1. ダッシュボード右下の **「メンバー管理」** ボタンをタップ
2. **「メンバー追加」** をクリック
3. 以下を入力して **「登録して招待メール送信」**

| 入力項目 | 内容 |
|----------|------|
| 氏名 | 例: 田中 太郎 |
| メールアドレス | ログインに使用するメールアドレス |
| 役職 | 担当職員 / 協力会社（火元責任者） |
| 会社名 | 所属会社名 |
| 作業所名 | 自分と同じ作業所名を入力（メール通知の紐付けに使用） |

4. 登録されたメンバーには招待メールが届く（初期パスワード記載、初回ログイン後に変更推奨）

**削除**
1. **「メンバー管理」** → 対象メンバーの **「削除」** をタップ
2. 確認ダイアログで **「削除」** → アカウントと申請データが削除される

> **注意**: 作業所名は所長・担当職員・協力会社で**完全一致**させてください。一致していないとメール通知が届きません。

---

#### ② 所長事前承認（正規ワークフロー）

**方法A（メールから直接承認）**
1. 「新規申請が届きました」のメールを開く
2. **「事前承認する →」** ボタンをクリック
3. ブラウザに「事前承認が完了しました」と表示されれば完了

**方法B（アプリから承認）**
1. メールの **「詳細を確認する →」** をクリックしてログイン
2. 申請内容を確認して **「事前承認」** をクリック
3. 指示事項を入力（デフォルト文あり）して **「承認」**

---

#### ① 事前承認（簡略ワークフロー）

1. 「新規申請が届きました」のメールを開く
2. **「① 事前承認する →」** ボタンをクリックしてアプリへ
3. 申請内容・現地確認チェック項目を確認
4. **「① 事前承認する →」** をクリック → 協力会社に通知メールが届く

> 担当職員が先に承認した場合は操作不要です。どちらか一方の承認で次のステップに進みます。

---

#### ⑦ 最終承認（正規・簡略共通）

1. 「残火確認完了 — 最終承認をお願いします」のメールが届いたらアプリにログイン
2. ダッシュボードの「未対応」の申請をタップ
3. **「最終承認」** をクリック → 協力会社に完了メールが届く

---

### 担当職員（supervisor）

#### ③-2 担当職員直前承認（正規ワークフローのみ・現地確認後）

1. 「現地確認完了」のメールが届いたらアプリにログイン
2. ダッシュボードの「未対応」の申請をタップ → **「直前承認」**
3. **「承認」** → 協力会社に作業開始可能のメールが届く

---

#### ④-2 作業中点検サイン（正規ワークフローのみ）

1. 「作業中点検サインをお願いします」のメールが届いたらアプリにログイン
2. ダッシュボードの「未対応」の申請をタップ → **「担当職員点検サイン」**
3. 点検時刻・残火確認時刻を入力してサイン → 協力会社に通知

---

#### ① 事前承認（簡略ワークフロー）

1. 「新規申請が届きました」のメールを開く
2. **「① 事前承認する →」** ボタンをクリックしてアプリへ
3. 申請内容・現地確認チェック項目（協力会社が提出時に記入）を確認
4. **「① 事前承認する →」** をクリック → 協力会社に通知メールが届く

> 所長が先に承認した場合は操作不要です。

---

#### ⑥ 残火確認（正規・簡略共通）

1. 「残火確認をお願いします」のメールが届いたらアプリにログイン
2. ダッシュボードの「未対応」の申請をタップ → **「残火確認へ」**
3. 残火確認時刻を入力して **「確認完了」** → 所長に最終承認依頼メールが届く

---

### 差し戻し（担当職員・所長共通）

- 申請内容に問題がある場合は各承認画面から **「差し戻し」** を選択
- コメントを入力して差し戻すと協力会社にメール通知
- 協力会社は内容を修正して再提出できる

---

### パスワード変更（全役職共通）

1. 画面右上のヘッダーメニュー（「…」または名前）をタップ
2. **「パスワード変更」** を選択
3. 新しいパスワードを入力して **「変更する」**

## ステータス一覧

### 正規ワークフロー

| ステータス | 意味 | 次のアクション担当 |
|-----------|------|-----------------|
| 下書き | 作成中 | 協力会社 |
| 所長事前確認待ち | 提出済み | 所長 |
| 現地確認待ち | 所長承認済み | 協力会社 |
| 担当職員直前承認待ち | 現地確認済み | 担当職員 |
| 作業中点検待ち | 直前承認済み | 協力会社 |
| 担当職員点検待ち | 火元責任者サイン済み | 担当職員 |
| 作業終了報告待ち | 担当職員サイン済み | 協力会社 |
| 残火確認待ち | 終了報告済み | 担当職員 |
| 最終承認待ち | 残火確認済み | 所長 |
| 承認済み | 完了 | — |
| 差し戻し | 要修正 | 協力会社 |

### 簡略ワークフロー

| ステータス | 意味 | 次のアクション担当 |
|-----------|------|-----------------|
| 下書き | 作成中 | 協力会社 |
| 所長事前確認待ち | 提出済み | 担当職員 または 所長 |
| 事前承認済み | 事前承認完了 | 協力会社（点検・終了報告） |
| 残火確認待ち | 終了報告済み | 担当職員 |
| 最終承認待ち | 残火確認済み | 所長 |
| 承認済み | 完了 | — |
| 差し戻し | 要修正 | 協力会社 |
